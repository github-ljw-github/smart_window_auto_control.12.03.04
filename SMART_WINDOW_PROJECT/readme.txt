值得注意的地方：
1. 工程代码优化等级为零，否则将可能出错。
2. 系统调度策略为时间片轮转调度，意味着某个任务不让出CPU将永远占用CPU
3. 使用微库MicroLib支持printf函数
4. 堆空间分配为0x8个字节，为支持更多的栈空间，基本使用不到标准库中的内存管理函数malloc etc.
5. 此系统是一个demo系统（演示系统），和实际应用有较大的距离。
系统功能：
1. 雨天自动关窗并启动雨刷清洁外窗
2. 雾霾天气自动关闭防雾霾纱窗
3. 室内发生有毒气体泄漏时自动打开窗户并声光报警、电话报警
4. 发现撬窗盗窃自动语音报警
5. 键盘长时间(8s)处于解锁状态且无有效操作则键盘自动上锁并关闭窗户的功能
6. 温度、湿度检测并实时显示于LCD上的功能，兼具高温报警功能。
7. 可通过键盘设置报警电话号码和报警温度临界值。
8. 自动判别动作的优先级，例如室内发生有毒气体的同时处于下雨天，则输出结果是打开窗户
系统缺陷：
1. 如果用户自己打开了窗户，然后外出了，磁控开关不能发现有人撬窗而入
2. 使用电机来驱动窗体的操作产生较大的噪音环境
3. 目前的demo板只能通过近距离的键盘操作控制智能窗，后续应当加入远程控制功能，如红外、蓝牙、WiFi等

启动流程：
1. 上电复位
2. 从地址0x00000000(映射地址为0x08000000)处取得栈顶的位置并保存SP寄存器（初始化栈）
3. 从地址0x00000004(映射地址为0x08000004)处取得复位向量的入口地址并保存到PC寄存器
4. 从复位向量的入口开始执行一系列操作，包括初始化系统时钟、初始化C/C++库（堆在这部分被初始化）、读写数据段的重定位
5. 从C/C++库跳转到main函数
6. 系统外设硬件等的初始化
7. 创建任务
8. FreeRTOS调度器启动，系统正式投入工作
